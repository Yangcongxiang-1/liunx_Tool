## 1. 什么是镜像源？

**镜像源（Mirror）** 是官方软件仓库的完整复制品，部署在不同地理位置。

形象理解：
> 官方源在国外 → 下载速度像「骑自行车去美国拿文件」
> 国内镜像源 → 文件就在「隔壁仓库」，秒级下载

镜像源的内容与官方源**完全同步**，只是服务器在国内，下载速度更快。

---

## 2. 为什么要配置镜像源？

| 场景 | 官方源速度 | 镜像源速度 |
|------|-----------|-----------|
| npm install | 50~200 KB/s | 5~50 MB/s |
| pip install | 经常超时 | 满速下载 |
| apt update | 几十分钟 | 几秒到几十秒 |
| docker pull | 经常断连 | 稳定高速 |

**核心好处：**
- ⚡ 下载速度提升 10~100 倍
- 🔒 避免网络超时/断连导致的安装失败
- 💰 节约等待时间
- 🌍 国内开发者必备

## 4. 支持的包管理器一览

本工具可配置 **16 种**包管理器的镜像源，外加 **Git 加速**：

| # | 包管理器 | 对应语言/生态 | 配置方式 | 需要 sudo？ |
|---|---------|-------------|---------|-----------|

| 2 | **pnpm** | Node.js (替代 npm) | `pnpm config set registry` | ❌ |
| 3 | **yarn** | Node.js (替代 npm) | `yarn config set registry` | ❌ |
| 4 | **bun** | Node.js (新一代) | 写入 `~/.bunfig.toml` | ❌ |
| 5 | **pip** | Python | 写入 `~/.pip/pip.conf` | ❌ |
| 6 | **gem** | Ruby | `gem sources --add` | ❌ |
| 7 | **bundler** | Ruby (依赖管理) | `bundle config mirror` | ❌ |
| 8 | **cargo** | Rust | 写入 `~/.cargo/config.toml` | ❌ |
| 9 | **rustup** | Rust (工具链管理) | 环境变量 `RUSTUP_DIST_SERVER` | ❌ |
| 10 | **go** | Go | `go env -w GOPROXY` | ❌ |
| 11 | **conda** | Python (数据科学) | `conda config` | ❌ |
| 12 | **composer** | PHP | `composer config -g` | ❌ |
| 13 | **apt** | Ubuntu/Debian | 写入 `/etc/apt/sources.list` | ✅ **必须** |
| 14 | **docker** | 容器 | 写入 `/etc/docker/daemon.json` | ✅ **必须** |
| 15 | **nvm** | Node.js 版本管理 | 环境变量 `NVM_NODEJS_ORG_MIRROR` | ❌ |
| 16 | **flutter/dart** | Flutter 应用 | 环境变量 `FLUTTER_STORAGE_BASE_URL` | ❌ |
| 17 | **homebrew** | macOS/Linux 包管理 | Git remote + 环境变量 | ❌ |
| 🔥 | **git** | GitHub 访问加速 | `url.insteadof` / HTTP 代理 | ❌ |

> 🔑 **需要 sudo 的配置**：apt 修改系统源列表、docker 写入系统配置，这两项必须用 `sudo` 执行。

## 5. 快速上手

### 5.1 交互菜单（最简单，推荐新手）

```bash
bash ~/桌面/set-mirrors.sh
```

进入菜单后：
1. 输入 `1` → 选择镜像源
2. 输入 `1`~`4` 选择源（默认阿里云，直接回车即可）
3. 等待自动配置完成 ✅

### 5.2 命令行一键设置（推荐熟悉命令行的用户）

```bash
# 阿里云（国内首选）
bash ~/桌面/set-mirrors.sh all

# 清华大学镜像
bash ~/桌面/set-mirrors.sh all 2

# 中科大镜像
bash ~/桌面/set-mirrors.sh all 3

# 华为云镜像
bash ~/桌面/set-mirrors.sh all 4

# 需要 sudo 的部分一起跑
sudo bash ~/桌面/set-mirrors.sh all
```

### 5.3 完整流程示例

```bash
# Step 1: 进入桌面
cd ~/桌面

# Step 2: 先看当前配置
bash set-mirrors.sh show

# Step 3: 一键设置所有镜像源（阿里云）
bash set-mirrors.sh all

# Step 4: 用 sudo 补充 apt 和 docker 配置
sudo bash set-mirrors.sh all

# Step 5: 重新加载配置
source ~/.bashrc
```

## 7. 命令行使用教程

### 7.1 基本语法

```bash
bash set-mirrors.sh <命令> [镜像源编号]
```

| 命令 | 说明 |
|------|------|
| `interactive` 或 `i` | 进入交互菜单 |
| `all` 或 `a` | 一键设置所有包管理器 |
| `show` 或 `status` | 查看当前镜像源配置 |
| `npm` / `pip` / `go` ... | 单独配置某个包管理器 |
| `help` | 查看帮助信息 |

### 7.2 镜像源编号

| 编号 | 镜像源 |
|------|--------|
| `1` (默认) | 阿里云 |
| `2` | 清华大学 TUNA |
| `3` | 中科大 USTC |
| `4` | 华为云 |
| `0` | 恢复官方源 |

### 7.3 使用示例

```bash
# 进入交互菜单
bash set-mirrors.sh i

# 查看当前配置
bash set-mirrors.sh show

# 一键设置所有（阿里云）
bash set-mirrors.sh all

# 一键设置所有（清华源）
bash set-mirrors.sh all 2

# 单独配置 pip（中科大）
bash set-mirrors.sh pip 3

bash set-mirrors.sh go 4

# apt 和 docker 需要 sudo
sudo bash set-mirrors.sh apt 2
sudo bash set-mirrors.sh docker 2

# 恢复 pip 到官方源
bash set-mirrors.sh pip 0
```

## 8. 单项配置教程

有时你只想配置某一个包管理器，而不是全部。以下是常见场景：

### 8.1 配置 npm 镜像

```bash
bash ~/桌面/set-mirrors.sh npm
```

验证是否成功：
```bash
npm config get registry
# 应输出: https://registry.npmmirror.com
npm cache clean --force   # 清理缓存
```

### 8.2 配置 pip 镜像

```bash

```

验证：
```bash
pip config list | grep index-url
# 或直接下载: pip install numpy
```

### 8.3 配置 apt 镜像（需要 sudo）

```bash
sudo bash ~/桌面/set-mirrors.sh apt
```

验证：
```bash
apt-cache policy | grep mirror
sudo apt update        # 更新源
```

### 8.4 配置 docker 镜像加速（需要 sudo）

```bash
sudo bash ~/桌面/set-mirrors.sh docker
```

验证：
```bash
docker info | grep -A 1 "Registry Mirrors"
docker pull nginx:latest  # 测试拉取速度
```

### 8.5 配置 go 代理

```bash
bash ~/桌面/set-mirrors.sh go
```

验证：
```bash
go env GOPROXY
```

### 8.6 🔥 配置 Git 加速

当 `git clone` GitHub 仓库只有几 KB/s 时，git 加速能解决问题。

脚本支持 **3 种模式**：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **ghproxy 加速** (推荐) | 通过 `git config url.insteadof` 自动将 GitHub 请求路由到国内代理 | 没有代理，单纯想加速 GitHub 访问 |
| **自定义 HTTP 代理** | 设置 `http.proxy` / `https.proxy` | 已有自己的代理（如 Clash、V2Ray） |
| **恢复官方** | 移除所有加速配置 | 不再需要加速 / 网络恢复正常 |

#### 8.6.1 ghproxy 加速（推荐）

```bash
# 使用 ghproxy 加速（默认 ghproxy.net）
bash ~/桌面/set-mirrors.sh git

# 或显式指定模式 1
bash ~/桌面/set-mirrors.sh git 1
```

运行后会让你选择 ghproxy 服务节点：
```
选择 ghproxy 服务:
  1) ghproxy.net
  2) ghproxy.com
  3) gh.api.99988866.xyz
```

**原理**：脚本在 git 全局配置中写入：

```bash
git config --global url."https://ghproxy.net/https://github.com/".insteadof "https://github.com/"
```

配置后，所有 `git clone https://github.com/xxx/xxx.git` 会自动改写为 `git clone https://ghproxy.net/https://github.com/xxx/xxx.git`，**无需修改任何命令**。

验证是否生效：
```bash
git config --global --get-regexp 'url\.'
# 应输出类似: url.https://ghproxy.net/...insteadof https://github.com/
```

测试加速效果：
```bash
# 加速前 VS 加速后
time git clone https://github.com/curl/curl.git --depth=1
```

#### 8.6.2 自定义 HTTP 代理

如果你已经有代理工具（如 Clash、V2Ray、SSR 等），可以直接让 git 走代理：

```bash
bash ~/桌面/set-mirrors.sh git 2
```

然后输入代理地址，例如：
```
请输入 HTTP 代理地址: http://127.0.0.1:7890
```

这会在 git 全局配置中写入：
```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

#### 8.6.3 SSH 代理（高级）

如果使用 SSH 协议连接 GitHub（`git@github.com:user/repo.git`），上面的方法不生效，需要配置 SSH 代理：

编辑 `~/.ssh/config`，添加：
```
Host github.com
  HostName ssh.github.com
  Port 443
  ProxyCommand nc -X connect -x 127.0.0.1:7890 %h %p
```

> 注：脚本暂不支持自动配置 SSH 代理，因为需要检测用户的代理类型和地址。

#### 8.6.4 恢复 git 官方直连

```bash
# 移除所有 git 加速配置
bash ~/桌面/set-mirrors.sh git 0
```

## 10. 恢复官方源

如果你需要恢复到原始官方源，使用编号 `0`：

```bash
# 一键全部恢复官方
bash ~/桌面/set-mirrors.sh all 0

# 单项恢复
bash ~/桌面/set-mirrors.sh npm 0
bash ~/桌面/set-mirrors.sh pip 0
sudo bash ~/桌面/set-mirrors.sh apt 0
```

> 注意：恢复官方源后，npm 会回到 `https://registry.npmjs.org`，pip 回到 `https://pypi.org/simple`，下载速度会变慢。

## 11. 注意事项

### ⚠️ 重要提醒

| # | 注意事项 | 说明 |
|---|---------|------|
| 1 | **apt 和 docker 需要 sudo** | `sudo bash set-mirrors.sh apt` 或 `sudo bash set-mirrors.sh all` |
| 2 | **配置后需重载 shell** | 执行 `source ~/.bashrc` 或 `source ~/.zshrc` 使环境变量生效 |
| 3 | **自动备份原文件** | 修改前会自动备份，备份文件带 `.bak.时间戳` 后缀 |
| 4 | **未安装的包自动跳过** | 不用担心报错，只会配置已安装的包管理器 |
| 5 | **环境变量重启后失效？** | 脚本已写入 `~/.bashrc`/`~/.zshrc`，重启后自动生效 |
| 6 | **镜像源可能延迟** | 镜像源与官方源有短暂同步延迟（通常几分钟~几小时） |
| 7 | **npm 缓存旧数据** | 更换 npm 源后建议运行 `npm cache clean --force` |
| 8 | **pip 的 trusted-host** | 脚本会自动配置 trusted-host，避免 HTTPS 警告 |
| 9 | **Docker 配置后需重启** | 脚本会自动重启 docker 服务 |

### 🔒 sudo 使用说明

apt 和 docker 的配置需要 root 权限：

```bash
# 方法一：全部一起跑（推荐）
sudo bash ~/桌面/set-mirrors.sh all

# 方法二：分开跑
bash ~/桌面/set-mirrors.sh all       # 先跑非 sudo 部分
sudo bash ~/桌面/set-mirrors.sh all   # 再跑 sudo 部分

# 方法三：只跑 sudo 的单项
sudo bash ~/桌面/set-mirrors.sh apt
sudo bash ~/桌面/set-mirrors.sh docker
```

> 交互模式下会自动检测并提示是否需要 sudo。

### 📂 备份文件位置

所有被修改的配置文件都会自动备份，备份格式为：
```
原文件名.bak.20260524_143021
```

备份文件在原文件同目录下，如果需要恢复：
```bash
# 例如恢复 pip 配置
mv ~/.pip/pip.conf.bak.* ~/.pip/pip.conf
```

## 12. 常见问题 FAQ

### Q1: 运行脚本报错 `Permission denied`

**原因：** 脚本没有执行权限。

**解决：**
```bash
chmod +x ~/桌面/set-mirrors.sh
```

### Q2: 某些包管理器显示「未安装，跳过」

**原因：** 你的系统没有安装对应的包管理器，这是正常行为。

**解决：** 先安装对应的包管理器，再运行脚本：
```bash
# 例如安装 pnpm
npm install -g pnpm

# 安装 go
sudo apt install golang

# 安装 rust/cargo
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

### Q3: 配置后 npm install 还是很慢？

**可能是缓存问题：**
```bash
npm cache clean --force
```

或验证 registry 是否真的变了：
```bash
npm config get registry
# 应该显示: https://registry.npmmirror.com
```

### Q4: apt 配置后 `apt update` 报错

可能是你的 Ubuntu 版本不在镜像源支持列表中：

```bash
# 查看你的版本
cat /etc/os-release

# 确认 codename
lsb_release -c
```

如果确实不支持，可以手动编辑 `/etc/apt/sources.list`。

### Q5: 配置了 docker 但拉取还是很慢

```bash
# 检查是否生效
docker info | grep "Registry Mirrors"

# 如果没有输出，重启 docker
sudo systemctl restart docker
```

### Q6: 如何只配置某一个源（比如只配 pip）？

```bash
bash ~/桌面/set-mirrors.sh pip
```

或进入交互菜单选「选项 2 → 单项配置」。

### Q7: 配置完需要重启电脑吗？

**不需要。** 大部分配置即时生效。环境变量相关的配置需要：
```bash
source ~/.bashrc   # 如果你用的是 bash
# 或
source ~/.zshrc    # 如果你用的是 zsh
```

### Q8: 不同镜像源哪个最快？

| 你的网络环境 | 推荐镜像源 |
|-------------|-----------|
| 电信/联通/移动宽带 | 阿里云 |
| 教育网 (CERNET) | 清华 / 中科大 |
| 华为云 / 企业内网 | 华为云 |
| 海外 | 官方源即可，不需要镜像 |
